## Phase 1: Critical Streaming Fixes

### Task 1: Fix camelCase field names in session_update parser

**Objective:** The ACP wire protocol sends camelCase (`toolCallId`,
`rawInput`, `rawOutput`) but our parser reads snake_case (`tool_call_id`).
This silently drops tool call events.

**Files:**
- Modify: `src/acp.rs` (handle_session_update)

**Steps:**

1. In `handle_session_update`, fix all field lookups in the `tool_call`
   and `tool_call_update` branches to check camelCase first, snake_case
   as fallback:

   - `tool_call_id` → check `toolCallId` then `tool_call_id`
   - `raw_input` → check `rawInput` then `raw_input`
   - `raw_output` → check `rawOutput` then `raw_output`

   For the `tool_call` branch:
   ```rust
   "tool_call" => {
       let id = params
           .get("toolCallId")
           .or_else(|| params.get("tool_call_id"))
           .and_then(|v| v.as_str())
           .unwrap_or("")
           .to_string();
       let name = params
           .get("title")
           .and_then(|v| v.as_str())
           .unwrap_or("")
           .to_string();
       let kind = params
           .get("kind")
           .and_then(|v| v.as_str())
           .map(|s| s.to_string());
       let _ = event_tx.send(AppEvent::ToolCallStart { id, name, kind });
   }
   ```

   For the `tool_call_update` branch:
   ```rust
   "tool_call_update" => {
       let id = params
           .get("toolCallId")
           .or_else(|| params.get("tool_call_id"))
           .and_then(|v| v.as_str())
           .unwrap_or("")
           .to_string();
       let status = params
           .get("status")
           .and_then(|v| v.as_str())
           .unwrap_or("")
           .to_string();
       let content = params
           .pointer("/content/0/content/text")
           .or_else(|| params.pointer("/content/0/text"))
           .and_then(|v| v.as_str())
           .map(|s| s.to_string());
       let _ = event_tx.send(AppEvent::ToolCallUpdate {
           id,
           status,
           content,
       });
   }
   ```

   Note: The tool_call_update content is nested as
   `content[0].content.text` (not `content[0].text`), because each
   content item is a `ContentToolCallContent` with its own `content`
   field containing a `TextContentBlock`.

2. Also fix the `prompt_done` event parsing — the ACP server doesn't
   send a separate `prompt_done` session_update. Instead, the `prompt()`
   RPC response itself signals completion. We need to emit PromptDone
   when the prompt request returns.

   In `app.rs`, update the spawned prompt task:
   ```rust
   tokio::spawn(async move {
       let result = acp.prompt(&prompt_text, &session_id).await;
       match result {
           Ok(val) => {
               let stop_reason = val
                   .get("stop_reason")
                   .or_else(|| val.get("stopReason"))
                   .and_then(|v| v.as_str())
                   .unwrap_or("end_turn")
                   .to_string();
               // Extract usage if present
               let usage = val.get("usage").and_then(|u| {
                   Some(Usage {
                       input_tokens: u.get("input_tokens")
                           .or_else(|| u.get("inputTokens"))
                           .and_then(|v| v.as_u64())?,
                       output_tokens: u.get("output_tokens")
                           .or_else(|| u.get("outputTokens"))
                           .and_then(|v| v.as_u64())?,
                   })
               });
               let _ = event_tx.send(AppEvent::PromptDone { stop_reason, usage });
           }
           Err(e) => {
               let _ = event_tx.send(AppEvent::AcpError(
                   format!("Prompt failed: {}", e),
               ));
           }
       }
   });
   ```

3. Verify: `cargo clippy --all-targets -- -D warnings`

4. Commit:
```bash
git add src/acp.rs src/app.rs
git commit -m "fix: camelCase field parsing for tool events, PromptDone from response"
```

## Phase 2: Input UX

### Task 3: Input history (up/down arrow)

**Objective:** Store previous user inputs and navigate them with
Up/Down arrows when the input field is active.

**Files:**
- Modify: `src/app.rs`

**Steps:**

1. Add history state to `App`:
   ```rust
   // In App struct:
   pub input_history: Vec<String>,
   pub history_index: Option<usize>,  // None = typing new, Some(i) = browsing
   pub saved_input: String,           // saves current input when browsing
   ```

2. Initialize in `App::new`:
   ```rust
   input_history: Vec::new(),
   history_index: None,
   saved_input: String::new(),
   ```

3. On Enter (when submitting a message), push to history:
   ```rust
   // Before clearing input:
   if !text.is_empty() && !text.starts_with('/') {
       self.input_history.push(text.clone());
   }
   self.history_index = None;
   self.saved_input.clear();
   ```

4. Handle Up arrow in chat mode (only when Idle and no modifier):
   ```rust
   (_, KeyCode::Up) if self.status == AgentStatus::Idle => {
       if self.input_history.is_empty() {
           return Ok(());
       }
       match self.history_index {
           None => {
               // Save current input, go to last history entry
               self.saved_input = self.input.clone();
               let idx = self.input_history.len() - 1;
               self.history_index = Some(idx);
               self.input = self.input_history[idx].clone();
               self.cursor = self.input.len();
           }
           Some(idx) if idx > 0 => {
               let idx = idx - 1;
               self.history_index = Some(idx);
               self.input = self.input_history[idx].clone();
               self.cursor = self.input.len();
           }
           _ => {}
       }
   }
   ```

5. Handle Down arrow:
   ```rust
   (_, KeyCode::Down) if self.status == AgentStatus::Idle => {
       match self.history_index {
           Some(idx) if idx + 1 < self.input_history.len() => {
               let idx = idx + 1;
               self.history_index = Some(idx);
               self.input = self.input_history[idx].clone();
               self.cursor = self.input.len();
           }
           Some(_) => {
               // Past the end — restore saved input
               self.history_index = None;
               self.input = std::mem::take(&mut self.saved_input);
               self.cursor = self.input.len();
           }
           None => {}
       }
   }
   ```

6. Add these arms BEFORE the existing `(_, KeyCode::Char(c))` arm in
   the chat key handler match block.

7. Verify: `cargo clippy --all-targets -- -D warnings`

8. Commit:
```bash
git add src/app.rs
git commit -m "feat: input history navigation with Up/Down arrows"
```

### Task 4: Multiline input (Shift+Enter / Alt+Enter)

**Objective:** Allow entering newlines in the input field. Regular Enter
submits; Shift+Enter or Alt+Enter inserts a newline.

**Files:**
- Modify: `src/app.rs` (handle_chat_key)
- Modify: `src/ui.rs` (draw_input)

**Steps:**

1. In `handle_chat_key`, add a Shift+Enter / Alt+Enter handler BEFORE
   the existing Enter handler:
   ```rust
   // Multiline: Shift+Enter or Alt+Enter inserts newline
   (KeyModifiers::SHIFT, KeyCode::Enter)
   | (KeyModifiers::ALT, KeyCode::Enter) => {
       self.input.insert(self.cursor, '\n');
       self.cursor += 1;
   }
   ```

2. Update `draw_input` in `ui.rs` to handle multiline display:
   - Count the number of lines in the input
   - Make the input box height dynamic (min 3, grows to fit)
   - Adjust the layout constraint from `Constraint::Length(3)` to
     `Constraint::Length(input_height)` where
     `input_height = 3.max(line_count + 2)` capped at some reasonable
     max (e.g., 10)

   In the main `draw_chat` layout:
   ```rust
   let input_lines = app.input.lines().count().max(1);
   let input_height = (input_lines as u16 + 2).min(10).max(3);
   ```

3. Verify: `cargo clippy --all-targets -- -D warnings`

4. Commit:

### Task 7: Ctrl+C cancels generation instead of quitting

**Objective:** When the agent is thinking, Ctrl+C should cancel the
current generation (via ACP cancel). Only quit if Ctrl+C is pressed
when idle, or use Ctrl+D / /quit.

**Files:**
- Modify: `src/app.rs` (handle_chat_key)

**Steps:**

1. Change the Ctrl+C handler in chat mode:
   ```rust
   (KeyModifiers::CONTROL, KeyCode::Char('c')) => {
       if self.status == AgentStatus::Thinking {
           // Cancel the current generation
           if let Some(sid) = &self.session_id {
               let acp = Arc::clone(acp);
               let sid = sid.clone();
               tokio::spawn(async move {
                   let _ = acp.cancel(&sid).await;
               });
           }
           self.sys_msg("Cancelled.");
           self.status = AgentStatus::Idle;
           // Flush any pending content
           let content = std::mem::take(&mut self.pending_response);
           if !content.is_empty() {
               self.messages.push(ChatMessage {
                   role: Role::Assistant,
                   content,
                   tokens: None,
               });
           }
           self.pending_thought.clear();
           self.active_tools.clear();
       } else {
           self.quit = true;
       }
   }
   ```

2. Keep Ctrl+D as always-quit:
   ```rust
   (KeyModifiers::CONTROL, KeyCode::Char('d')) => {
       self.quit = true;
   }
   ```

3. Verify: `cargo clippy --all-targets -- -D warnings`
