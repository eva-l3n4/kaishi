### A1: `list_sessions` enrichment — expose title, timestamps, source

**Objective:** The session picker currently shows only `session_id` and

expose it.

**Files:**
- Modify: `acp_adapter/server.py` (list_sessions method)
- Modify: `acp_adapter/session.py` (SessionManager.list_sessions)

**Steps:**

1. In `session.py`, update `list_sessions()` to return richer dicts
   from the DB rows:
   ```python
   # In the DB merge section, add:
   results.append({
       "session_id": sid,
       "cwd": cwd,

       "history_len": row.get("message_count") or 0,
       "title": row.get("title"),
       "started_at": row.get("started_at"),
       "last_active": row.get("last_active"),
       "source": row.get("source"),
   })
   ```

   For in-memory sessions, also include the DB metadata if available:
   ```python

   for s in self._sessions.values():

               info["last_active"] = row.get("last_active")
       results.append(info)
   ```

### A2: `get_session_history` — new ACP method to fetch messages

**Objective:** When resuming a session, the TUI has no way to display
past messages. Add a method that returns the conversation history.

**Files:**
- Modify: `acp_adapter/server.py`
- Modify: `acp_adapter/session.py`

**Steps:**

1. This is tricky because the ACP protocol schema doesn't define a
   `get_session_history` method. Two approaches:

   **Option A: Extension method** — register a custom method name
   (e.g., `hermes/get_session_history`) via the router's extension
   handler. The TUI sends a JSON-RPC request, the server responds
   with the message list.

   **Option B: Slash command** — add `/history` that returns the last
   N messages as formatted text. Simpler, no protocol extension needed,
   but loses structured data.

   Recommend **Option A** for structured history:

   ```python
   # In server.py, add to on_connect or as an extension handler:
   @router.handle_extension_request
   async def _handle_extension(name, payload):
       if name == "hermes/get_session_history":
           session_id = payload.get("session_id", "")
           limit = payload.get("limit", 50)
           state = self.session_manager.get_session(session_id)
           if state is None:
               return {"messages": []}
           messages = state.history[-limit:]
           # Simplify for wire: just role + content
           return {"messages": [
               {"role": m.get("role", ""), "content": str(m.get("content", ""))}
               for m in messages
           ]}
   ```

2. On the TUI side (`acp.rs`), add `get_session_history()`:
   ```rust
   pub async fn get_session_history(&self, session_id: &str, limit: usize)
       -> Result<Vec<(String, String)>>
   {
       let result = self.request(
           "hermes/get_session_history",
           Some(json!({"session_id": session_id, "limit": limit})),
       ).await?;
       // parse messages array
   }
   ```

3. Call it after `SessionResumed` event instead of `/context`.

### A3: `/title` slash command

**Objective:** Let the user name their sessions from the TUI.

**Files:**
- Modify: `acp_adapter/server.py`

**Steps:**

1. Add to `_SLASH_COMMANDS`:
   ```python
   "title": "Set or show session title",
   ```

2. Add to `_ADVERTISED_COMMANDS`:
   ```python
   {"name": "title", "description": "Set or show session title",
    "input_hint": "new title"},
   ```

3. Add handler:
   ```python
   def _cmd_title(self, args: str, state: SessionState) -> str:
       db = self.session_manager._get_db()
       if not args:
           if db:
               row = db.get_session(state.session_id)
               title = row.get("title") if row else None
               return f"Session title: {title or '(untitled)'}"
           return "Session title: (untitled)"
       if db:
           db.rename_session(state.session_id, args.strip())
       return f"Session title set: {args.strip()}"
   ```

4. Register in handler dict:
   ```python
   "title": self._cmd_title,
   ```

### A4: `/yolo` toggle

**Objective:** Toggle approval mode from within the TUI.

**Files:**
- Modify: `acp_adapter/server.py`

**Steps:**

1. Add to `_SLASH_COMMANDS` and handler dict.

2. Handler:
   ```python
   def _cmd_yolo(self, args: str, state: SessionState) -> str:
       agent = state.agent
       current = getattr(agent, "approval_mode", "manual")
       if current == "yolo" or current == "off":
           agent.approval_mode = "manual"
           return "Approval mode: manual (dangerous commands need approval)"
       else:
           agent.approval_mode = "yolo"
           return "Approval mode: yolo (all commands auto-approved)"
   ```

   Note: this sets it on the agent instance only, not the config file.
   It resets on session restart.
