1. Animation System — Dual-Rate Ticks

Two tick rates in the event loop. Content tick handles UI refresh. Animation tick drives spinner + shimmer at 50ms (20fps) without invalidating the message line cache.

pub enum AppEvent {
  // ... existing variants ...
  Tick,           // ~200ms — UI refresh, cursor blink
  AnimationTick,  // ~50ms — spinner frame, shimmer position, elapsed time
}
AnimationTick only bumps counters in AnimationState and triggers a re-render. It never touches ACP state, never invalidates the line cache, never rebuilds message Spans. The draw function reads AnimationState to render the spinner line — that single line is rebuilt each frame, everything else comes from cache.

2. AnimationState Struct

pub struct AnimationState {
  // Spinner
  pub frame: usize,                 // index into SPINNER_GLYPHS
  pub phase: AgentPhase,            // Thinking | Streaming | Executing | Idle
  pub phase_start: Instant,         // when current phase began
  pub last_output: Instant,         // last time we received content (for stall detection)

  // Shimmer (traveling highlight)
  pub shimmer_pos: usize,          // which char in the label is highlighted
  pub shimmer_tick: u8,            // sub-counter — advance shimmer every N animation ticks

  // Executing tool context
  pub active_tool: Option<String>,  // name of tool currently running
}

4. Shimmer — Traveling Highlight

A single bright character travels across the phase label text, with adjacent characters getting a half-brightness lift. Pure ANSI — no RGB.

✶  thinking · 3s

Color Mapping (ANSI-native)

// Three brightness levels using terminal-native colors:
const SHIMMER_DIM: Color = Color::DarkGray;  // base text
const SHIMMER_MID: Color = Color::Gray;     // adjacent to highlight
const SHIMMER_HI: Color = Color::White;    // the highlighted char

// Per character:
fn shimmer_color(char_idx: usize, highlight_pos: usize) -> Color {
  let dist = char_idx.abs_diff(highlight_pos);
  match dist {
    0 => SHIMMER_HI,
    1 => SHIMMER_MID,
    _ => SHIMMER_DIM,
  }
}
Shimmer advances every ~150ms (every 3rd animation tick at 50ms). The highlight wraps around: after the last character, it jumps back to the first. This matches CC's ShimmerChar pattern but with ANSI palette instead of RGB interpolation.

5. Turn Completion Summary

When the prompt response arrives with stopReason and usage, show a compact summary:

◆ Here's the refactored module. I've extracted the...

── 1,234 in · 567 out · 12.3s ──
Shown as a dim divider line after the assistant's final message. Only appears when we have usage data (which we always do from the prompt response). Format: input_tokens in · output_tokens out · elapsed. Elapsed is total turn time (from prompt sent to response received), not per-phase.

6. Spinner Glyph Details

· ✢ ✳ ✶ ✻ ✽
bounce: forward → reverse → forward ...
// Build the bounce sequence once:
const GLYPHS: &[char] = &['·', '✢', '✳', '✶', '✻', '✽'];

fn bounce_sequence() -> Vec<char> {
  let mut seq: Vec<char> = GLYPHS.to_vec();
  seq.extend(GLYPHS.iter().rev().skip(1).take(GLYPHS.len() - 2));
  seq // [· ✢ ✳ ✶ ✻ ✽ ✻ ✶ ✳ ✢]
}
